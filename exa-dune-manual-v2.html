1. Introduzione e Concetti Base

1.1 Cos'è exa-dune?

exa-dune è un framework di penetration testing scritto in Bash, progettato specificamente per ambienti IoT (Internet of Things) e VoIP (Voice over IP). È uno strumento a riga di comando che integra e coordina numerosi tool di sicurezza già presenti su Kali Linux, presentandoli attraverso un'interfaccia unificata e semplice.

Cos'è il Penetration Testing?
Il penetration testing (o "pentest") è il processo autorizzato di testare la sicurezza di un sistema informatico simulando gli attacchi di un hacker reale. L'obiettivo è trovare le vulnerabilità prima che lo facciano i malintenzionati, per poi poterle correggere.

1.2 Cosa può fare exa-dune?

  • Scansione di rete — Trova dispositivi attivi, porte aperte e servizi esposti
  • Brute-force delle credenziali — Testa username e password su molti protocolli
  • Audit di protocolli specifici — SIP/VoIP, SNMP, MQTT, Modbus, RTSP e altri
  • Analisi web — Trova directory nascoste, vulnerabilità LFI, header mancanti
  • Analisi firmware — Estrae e analizza firmware di dispositivi IoT
  • Pipeline automatizzata — Esegue scan → analisi → attacchi → report in sequenza
  • Generazione report PDF — Produce report professionali pronti per la documentazione

1.3 Concetti Fondamentali

Porta di Rete

Una porta è come una "porta d'ingresso" virtuale su un dispositivo di rete. Ogni servizio usa una porta specifica: il web usa la porta 80 (HTTP) o 443 (HTTPS), SSH usa la 22, VoIP/SIP usa la 5060. Esistono 65535 porte; alcune sono standardizzate, altre usate liberamente dai produttori.

Protocollo

Un protocollo è il "linguaggio" che due dispositivi usano per comunicare. SSH è un protocollo sicuro per accesso remoto; SIP è il protocollo per le telefonate VoIP; SNMP serve a gestire dispositivi di rete da remoto.

Brute Force

Il brute force consiste nel provare molte combinazioni di username e password fino a trovare quella corretta. È efficace quando si usano password deboli o di default (come "admin/admin"). exa-dune include oltre 119 password e 43 username pre-integrati.

Wordlist

Una wordlist è una lista di parole (username, password, percorsi web, community string) usata durante gli attacchi automatizzati. exa-dune v2.4.0 include 7 wordlist integrate direttamente nel codice, quindi funziona anche senza SecLists installato.

Avviso Legale
exa-dune deve essere utilizzato esclusivamente su sistemi di cui si è proprietari o per cui si ha un'esplicita autorizzazione scritta. L'uso non autorizzato è illegale in Italia (art. 615-ter, 617-quater, 640-ter c.p.) e nella maggior parte dei paesi. Questo strumento è destinato a laboratori personali isolati.

6. Subcomandi di Scansione e Ricognizione

scan
Scansione porte e servizi con nmap — punto di partenza per ogni engagement

Cos'è una scansione di porte?

Quando exa-dune "scansiona" un target, invia pacchetti TCP/UDP a ogni porta (da 1 a 65535) e analizza la risposta. Se la porta risponde, il servizio è attivo. nmap (il tool usato internamente) è in grado di rilevare anche quale software gira su quella porta e la sua versione, informazione fondamentale per trovare vulnerabilità note (CVE).

Sintassi

exa-dune scan --target TARGET [opzioni]

Flag Specifici

FlagDefaultDescrizione
--ports / -ptop-1000Porte da scansionare (es: 80,443,8080 oppure 1-65535)
--scan-typeSVTipo scan: SV (version), A (aggressive), sU (UDP)
--script—NSE script nmap aggiuntivi (es: vuln, auth, banner)
--timing / -TT4Velocità: T1(lento)–T5(veloce). T4 default, T2 con --stealth
--top-ports—Scansiona solo le N porte più comuni (es: --top-ports 100)
--udp—Aggiunge scansione UDP (richiede root, molto più lenta)
--ping-sweep—Solo ping, non scansione porte (rapido discovery)

Esempi Pratici

# Scan base di una sottorete (trova tutti gli host)
exa-dune scan --target 192.168.1.0/24 --output-dir /root/pentest/scans

# Scan aggressivo su singolo IP (rileva OS, versioni, script)
exa-dune scan --target 192.168.1.50 --scan-type A --output-dir /root/pentest/scans

# Scan furtivo (lento, evita IDS)
exa-dune scan --target 192.168.1.50 --stealth --output-dir /root/pentest/scans

# Scan solo porte VoIP/SIP
exa-dune scan --target 192.168.1.0/24 --ports 5060,5061,5080,1720,4569

# Scan UDP per SNMP e TFTP
exa-dune scan --target 192.168.1.50 --udp --ports 161,69,123,514

dns
Enumerazione DNS — trova sottodomini e record di un dominio

Cos'è l'enumerazione DNS?

Il DNS (Domain Name System) traduce nomi come voip.azienda.local in indirizzi IP. Enumerare il DNS significa scoprire tutti i sottodomini e i record esistenti, rivelando servizi nascosti che potrebbero non essere visibili dalla scansione di rete diretta.

Flag Specifici

FlagDescrizione
--domainDominio da enumerare (es: azienda.local)
--dns-serverServer DNS da interrogare (default: resolver di sistema)
--wordlistWordlist sottodomini personalizzata
--axfrTenta zone transfer (AXFR) — rivela tutti i record se abilitato
--bruteAbilita brute force sottodomini con wordlist integrata (57 voci)
# Enumerazione base con zone transfer
exa-dune dns --target 192.168.1.1 --domain lab.local --axfr

# Brute force sottodomini con wordlist integrata
exa-dune dns --target 192.168.1.1 --domain lab.local --brute

# Usa server DNS specifico
exa-dune dns --target 192.168.1.1 --domain lab.local --dns-server 192.168.1.1 --axfr
Zone Transfer (AXFR)
Un zone transfer è una funzionalità del protocollo DNS che permette di copiare tutti i record di un dominio da un server all'altro. Se il server DNS è mal configurato e accetta richieste AXFR da qualsiasi host, rivela l'intera mappa della rete interna. Questo è un problema comune nei laboratori e nelle reti aziendali non aggiornate.

7. Subcomandi VoIP e Multimedia

sip
Enumerazione SIP, discovery utenti, crack PIN — protocollo VoIP principale

Cos'è il protocollo SIP?

SIP (Session Initiation Protocol) è il protocollo standard per le telefonate VoIP. Funziona sulla porta 5060/UDP (o TCP) e 5061/TCP per SIP-TLS. Ogni telefono VoIP, gateway, PBX (centralino) e softphone usa SIP. Il protocollo gestisce la segnalazione: avvio, modifica e chiusura delle chiamate.

Un server SIP espone account/estensioni (es: 100, 101, 200) protetti da PIN. L'enumerazione SIP scopre quali estensioni esistono; il cracking testa i PIN comuni.

Flag Specifici

FlagDefaultDescrizione

--transportudpProtocollo: udp, tcp, tls
--enum—Enumera estensioni/utenti (usa svwar)
--crack—Tenta cracking PIN (usa svcrack)
--extension-range100-999Range estensioni da enumerare
--methodREGISTERMetodo SIP: REGISTER, OPTIONS, INVITE
--user-agentexa-duneUser-Agent SIP personalizzato
--domainIP targetDominio SIP (realm)

Esempi Pratici

# Discovery: verifica se il server risponde a SIP OPTIONS
exa-dune sip --target 192.168.1.50

# Enumerazione estensioni 100-500
exa-dune sip --target 192.168.1.50 --enum --extension-range 100-500

# Cracking PIN su estensione specifica
exa-dune sip --target 192.168.1.50 --crack --user 200

# SIP su TCP (AudioCodes spesso usa TCP)
exa-dune sip --target 192.168.1.50 --transport tcp --enum

# SIP-TLS (porta 5061)
exa-dune sip --target 192.168.1.50 --port 5061 --transport tls

Cos'è RTSP?

8. Subcomandi Brute-Force e Credenziali

Come Funziona il Brute Force in exa-dune
Tutti i comandi di brute force usano hydra internamente. hydra prova ogni combinazione username/password dalla wordlist e riporta le credenziali valide. La velocità dipende dal numero di thread (--threads). Con --stealth si usa 1 thread e si aggiungono delay per eludere i sistemi di rilevamento.
ssh
Brute force SSH — accesso remoto sicuro alla shell

SSH (Secure Shell) è il protocollo standard per accesso remoto sicuro. Porta default: 22/TCP. Molti dispositivi IoT/VoIP hanno SSH abilitato con credenziali di default o deboli.

# Brute force con wordlist integrate
exa-dune ssh --target 192.168.1.50 --output-dir /root/pentest/scans

# Test credenziali specifiche
exa-dune ssh --target 192.168.1.50 --user root --pass admin

# Wordlist personalizzata, 8 thread
exa-dune ssh --target 192.168.1.50 \
         --user-list /usr/share/seclists/Usernames/top-usernames-shortlist.txt \
         --pass-list /usr/share/seclists/Passwords/xato-net-10-million-passwords-10000.txt \
         --threads 8

# Output di successo
[+] FOUND: root:admin @ 192.168.1.50:22
[+] Credenziali salvate in: /root/pentest/scans/ssh_creds.txt
ftp
Brute force FTP — file transfer non cifrato

FTP (File Transfer Protocol) porta 21/TCP. Non cifra i dati — credenziali in chiaro. Molti dispositivi embedded usano FTP per aggiornamenti firmware o backup configurazione.

# Brute force FTP + test anonymous login
exa-dune ftp --target 192.168.1.50

# Test solo accesso anonimo
exa-dune ftp --target 192.168.1.50 --user anonymous --pass anonymous
telnet
Brute force Telnet — protocollo legacy non cifrato

Telnet (porta 23/TCP) è il precursore di SSH: accesso remoto ma senza cifratura. Ancora molto comune su dispositivi IoT/embedded economici e hardware industriale legacy.

# Brute force Telnet
exa-dune telnet --target 192.168.1.50

# Porta non standard
exa-dune telnet --target 192.168.1.50 --port 2323
Attenzione
Telnet trasmette username e password in chiaro. Se sei su una rete condivisa, le credenziali possono essere catturate con Wireshark da qualsiasi altro host. Considera sempre tcpdump/Wireshark per intercettare il traffico durante i test.
creds
Test credenziali di default IoT/VoIP su tutti i protocolli

Il subcomando creds è specializzato per dispositivi IoT/VoIP: usa la wordlist integrata _wl_iot_creds() con 65 coppie user:pass specifiche per vendor, e le testa automaticamente su tutti i protocolli rilevati (SSH, Telnet, FTP, HTTP, SIP). 

# Test tutte le credenziali IoT su tutti i protocolli

# Solo HTTP con credenziali vendor-specifiche

rdp
Brute force RDP — Remote Desktop Protocol (Windows)

RDP (porta 3389/TCP) permette il controllo grafico remoto di sistemi Windows. Raro sui dispositivi VoIP puri, ma presente su PC embedded con software gestione VoIP.

# Brute force RDP
exa-dune rdp --target 192.168.1.50 --output-dir /root/pentest/scans
vnc
Brute force VNC — accesso desktop remoto multipiattaforma

VNC (porta 5900/TCP) è un sistema di controllo remoto grafico. Comune su dispositivi embedded Linux e alcuni apparati di rete con interfaccia grafica.

# Brute force VNC
exa-dune vnc --target 192.168.1.50
smb
Enumerazione e brute force SMB — condivisioni file Windows/Samba

SMB (Server Message Block, porta 445/TCP) è usato per condivisioni file Windows e Samba su Linux. Vulnerabilità storiche: EternalBlue (MS17-010), WannaCry, NotPetya.

# Enumerazione condivisioni (null session)
exa-dune smb --target 192.168.1.50

9. Subcomandi Protocolli Industriali/IoT

snmp
SNMP community brute force e MIB walk — gestione dispositivi di rete

Cos'è SNMP?

SNMP (Simple Network Management Protocol) è il protocollo usato per monitorare e gestire dispositivi di rete (router, switch, VoIP gateway, UPS). Funziona su porta 161/UDP. L'accesso è controllato da una community string (simile a una password). Le community di default sono public (lettura) e private (scrittura).

Con la community string corretta, è possibile leggere la MIB (Management Information Base): un albero di informazioni sul dispositivo che include configurazione, routing table, utenti, interfacce di rete e persino le password in testo chiaro su alcuni dispositivi legacy.

mqtt
Subscribe/publish MQTT — protocollo IoT per messaggistica

Cos'è MQTT?

MQTT (Message Queuing Telemetry Transport) è un protocollo leggero publish/subscribe usato massivamente in ambienti IoT (sensori, automazione industriale, smart home). Porta default: 1883/TCP (non cifrato) o 8883/TCP (TLS). Un broker MQTT non autenticato permette di intercettare tutti i messaggi dell'impianto. 

# Discovery e subscribe a tutti i topic
exa-dune mqtt --target 192.168.1.50

# Subscribe a topic specifico
exa-dune mqtt --target 192.168.1.50 --topic "#"

# Publish messaggio su topic
exa-dune mqtt --target 192.168.1.50 --publish --topic "/test" --message "hello"

# Con autenticazione
exa-dune mqtt --target 192.168.1.50 --user admin --pass admin
modbus
Lettura registri Modbus/TCP — protocollo PLC industriale

Modbus/TCP (porta 502/TCP) è il protocollo usato in ambienti industriali per controllare PLC (Programmable Logic Controllers), sensori e attuatori. Non ha autenticazione. La lettura dei registri rivela lo stato dell'impianto; la scrittura può modificarlo. 

# Leggi registri Modbus (holding registers)
exa-dune modbus --target 192.168.1.50

# Leggi unit ID specifico e range registri
exa-dune modbus --target 192.168.1.50 --unit-id 1 --register 0-100
Attenzione — Sistemi Industriali
Scrivere su registri Modbus in un impianto reale può causare malfunzionamenti fisici. Eseguire SOLO in ambienti di laboratorio con dispositivi non critici.
redis
Accesso a Redis non autenticato — database in-memory

Redis (porta 6379/TCP) è un database chiave-valore in memoria. Spesso deployato senza autenticazione in reti interne. Un Redis esposto permette la lettura di tutti i dati e può essere usato per escalation (es: scrittura chiavi SSH per accesso root).

# Test accesso non autenticato + dump keys
exa-dune redis --target 192.168.1.50

# Tenta escalation via authorized_keys
exa-dune redis --target 192.168.1.50 --exploit
ldap
Enumerazione directory LDAP/Active Directory

LDAP (porta 389/TCP, o 636 per LDAPS) è il protocollo per directory aziendali (Active Directory, OpenLDAP). Permette di enumerare utenti, gruppi, configurazioni di rete.

# Enumerazione LDAP con null bind (anonimo)
exa-dune ldap --target 192.168.1.50

# Con credenziali
exa-dune ldap --target 192.168.1.50 --user "cn=admin,dc=lab,dc=local" --pass admin
upnp
Discovery UPnP — Universal Plug and Play

UPnP (porta 1900/UDP) permette ai dispositivi di scoprirsi e configurarsi automaticamente sulla rete. Spesso espone informazioni dettagliate sul dispositivo e può permettere l'apertura di porte sul router.

# Discovery UPnP sulla rete
exa-dune upnp --target 192.168.1.0/24

# Descrizione XML del dispositivo specifico
exa-dune upnp --target 192.168.1.1 --describe
tftp
Download file via TFTP — protocollo di trasferimento senza autenticazione

TFTP (Trivial FTP, porta 69/UDP) è usato da molti dispositivi embedded per il boot di rete (PXE) e aggiornamenti firmware. Non ha autenticazione e permette il download di file di configurazione sensibili se mal configurato.

# Tenta download di file di configurazione comuni
exa-dune tftp --target 192.168.1.50

# Download file specifico
exa-dune tftp --target 192.168.1.50 --file startup-config
coap
CoAP — Constrained Application Protocol per IoT

CoAP (porta 5683/UDP) è il protocollo HTTP-like per dispositivi IoT a bassa potenza. Permette GET/PUT/POST su risorse del dispositivo. Raro ma presente su sensori industriali moderni.

# Discovery risorse CoAP
exa-dune coap --target 192.168.1.50
ipmi
IPMI/BMC — gestione server out-of-band

IPMI (porta 623/UDP) è usato per la gestione remota di server (BMC/iDRAC/iLO). La vulnerabilità IPMI Cipher 0 permette l'autenticazione con qualsiasi password. Molti BMC hanno credenziali di default admin:admin o ADMIN:ADMIN.

# Dump hash password IPMI (Cipher 0 bypass)
exa-dune ipmi --target 192.168.1.50

# Brute force IPMI
exa-dune ipmi --target 192.168.1.50 --brute

11. Pipeline Automatizzata

FUNZIONALITÀ v2.4.0
La pipeline automatizzata è la funzionalità più potente di exa-dune: esegue l'intero ciclo di penetration testing in un singolo comando, dalla scansione iniziale al report PDF finale.

11.1 Le 5 Fasi della Pipeline

FASE 1

Scansione

Esegue nmap sul target con rilevamento versioni e script NSE. Salva i risultati in formato .nmap, .xml e .gnmap per l'analisi automatica. Con --from-scan DIR questa fase viene saltata e si usano scan precedenti.

FASE 2

Analisi e Suggerimenti

Legge l'output nmap e mappa le porte aperte ai moduli di attacco appropriati. Ogni suggerimento ha una priorità da 1 (critico) a 25 (basso). La lista viene mostrata ordinata per priorità con codice colore: rosso=critico, giallo=medio, verde=basso.

FASE 3

Esecuzione Attacchi

Esegue ogni attacco suggerito in ordine di priorità. Per ogni attacco registra l'output completo. Se non vengono trovate vulnerabilità, logga "nessuna vulnerabilità rilevata". Con --auto non chiede conferma prima di ogni attacco.

FASE 4

Post-Exploitation Automatico

Se hydra ha trovato credenziali valide, le legge automaticamente e lancia exa-dune post per raccogliere informazioni dal sistema compromesso. Con --no-post questa fase viene saltata.

FASE 5

Generazione Report PDF

Aggrega tutti gli output delle fasi precedenti in un report HTML strutturato e lo converte in PDF con weasyprint. Il report include executive summary, tabella vulnerabilità, evidenze e raccomandazioni.

11.2 Sintassi Completa

exa-dune pipeline --target TARGET [opzioni]

11.3 Flag Specifici Pipeline

FlagDefaultDescrizione
--target / -t—Target: IP singolo, CIDR, o hostname
--output-dir / -o./pipeline_TIMESTAMPDirectory output per tutti i file
--auto—Esegui senza chiedere conferma ad ogni fase
--from-scan DIR—Usa scan nmap esistente, salta Fase 1
--no-post—Salta Fase 4 (post-exploitation)
--stealth—Modalità silenziosa in tutte le fasi
--timeout N300Timeout in secondi per ogni singolo attacco
--dry-run—Simula tutto senza eseguire azioni reali
--pdf—Genera PDF (attivo di default se weasyprint disponibile)

11.4 Esempi Pratici

━━━ UTILIZZO BASE ━━━
# Pipeline completa su target singolo
exa-dune pipeline --target 192.168.1.50 --output-dir /root/pentest/scans

━━━ MODALITÀ AUTOMATICA ━━━
# Nessuna interazione richiesta, ideale per script
exa-dune pipeline --target 192.168.1.50 --auto --output-dir /root/pentest/scans

━━━ RIUSO SCAN ESISTENTE ━━━
# Salta la scansione, usa risultati precedenti
exa-dune pipeline --from-scan /root/pentest/scans/scan_20260321/ \
         --auto --output-dir /root/pentest/scans

━━━ SILENZIOSA SENZA POST-EXPLOIT ━━━
exa-dune pipeline --target 192.168.1.50 \
         --stealth --no-post \
         --timeout 600 \
         --output-dir /root/pentest/scans

━━━ DRY RUN (test senza azioni) ━━━
exa-dune pipeline --target 192.168.1.50 --dry-run

13. Glossario per Principianti

Definizioni dei termini tecnici usati in questo manuale, con spiegazioni accessibili.

Banner Grabbing
Tecnica per leggere il "banner" (messaggio di benvenuto) di un servizio che rivela la versione del software. Es: SSH-2.0-OpenSSH_7.4. Utile per trovare versioni vulnerabili.
Brute Force
Attacco che prova sistematicamente molte combinazioni di credenziali (username/password) fino a trovare quella corretta. Contrapposto al "dictionary attack" che usa una lista di parole predefinite.
CGI (Common Gateway Interface)
Standard per eseguire programmi sul server web. Le URL che contengono /cgi-bin/ spesso eseguono script che possono avere vulnerabilità di injection.
CIDR (Classless Inter-Domain Routing)
Notazione per specificare range di indirizzi IP. 192.168.1.0/24 indica tutti gli indirizzi da 192.168.1.0 a 192.168.1.255 (256 host). /24 indica 24 bit di network.
Community String (SNMP)
La "password" del protocollo SNMP. Le più comuni sono public (sola lettura) e private (lettura/scrittura). Molti dispositivi le lasciano al valore di default.
CVE (Common Vulnerabilities and Exposures)
Sistema di identificazione standard per vulnerabilità note. Formato: CVE-ANNO-NUMERO (es: CVE-2021-44228 = Log4Shell). Ogni CVE è una vulnerabilità documentata pubblicamente.
CVSS (Common Vulnerability Scoring System)
Sistema di valutazione della gravità delle vulnerabilità da 0.0 a 10.0. Versione 3.1 è quella attuale. Considera vettore d'attacco, complessità, privilegi richiesti, impatto. Score ≥9.0 = critico.
Default Credentials
Username e password preimpostate in fabbrica (es: admin/admin). Sono la prima cosa da cambiare su qualsiasi dispositivo. La maggior parte delle compromissioni IoT sfruttano credenziali di default non cambiate.
Directory Traversal / Path Traversal
Vulnerabilità che permette di accedere a file fuori dalla directory web usando ../. Esempio: http://victim/page?file=../../../etc/passwd. Anche chiamata LFI (Local File Inclusion).
Dry Run
Modalità di esecuzione che mostra i comandi che verrebbero eseguiti senza eseguirli realmente. Utile per verificare cosa farebbe lo strumento prima di procedere.
Enumeration
Fase di ricognizione che raccoglie informazioni specifiche su un target: utenti, directory, servizi, condivisioni. Più dettagliata del semplice scan di porte.
Exploit
Codice o tecnica che sfrutta una vulnerabilità per ottenere un accesso non autorizzato o eseguire azioni non previste. Un exploit "in the wild" è già usato da attaccanti reali.
Extension (SIP)
Identificatore numerico (solitamente 3-4 cifre) di un account SIP. Equivale al "numero interno" di un centralino. Es: estensione 200 = telefono IP 192.168.1.50, account 200.
Firmware
Software permanente installato nella memoria flash di un dispositivo hardware. Controlla il funzionamento del dispositivo. I firmware obsoleti spesso contengono vulnerabilità note.
Fuzzing
Tecnica di testing che invia input casuali o sistematici a un programma per trovare comportamenti inattesi (crash, errori, accessi non autorizzati). Il web fuzzing cerca percorsi URL nascosti.
Gateway VoIP
Dispositivo che converte segnale telefonico analogico (PSTN) in VoIP digitale. Es: AudioCodes MP202 converte telefoni analogici in chiamate SIP su Internet.
hydra
Tool di brute force per reti, open source, incluso in Kali Linux. Supporta oltre 30 protocolli: SSH, FTP, HTTP, SIP, RDP, VNC e molti altri. Usato internamente da exa-dune per brute force.
IoT (Internet of Things)
Dispositivi fisici connessi a Internet: telecamere, sensori, router, gateway VoIP, elettrodomestici "smart". Spesso hanno sicurezza debole per ridurre i costi di produzione.
LFI (Local File Inclusion)
Vulnerabilità web che permette di leggere file locali del server attraverso un parametro URL manipolato. Target comune: /etc/passwd, /etc/shadow, file di configurazione.
MIB (Management Information Base)
Database gerarchico di informazioni gestibili via SNMP. Ogni nodo ha un OID (Object Identifier) numerico. La MIB di un dispositivo espone tutto: configurazione, statistiche, utenti.
nmap
Network Mapper — tool open source per scansione di rete. Uno dei più usati in sicurezza informatica. Rileva host attivi, porte aperte, versioni dei servizi e può eseguire script NSE di sicurezza.
NSE (Nmap Scripting Engine)
Sistema di scripting di nmap che permette di eseguire analisi avanzate. Script come vuln testano vulnerabilità note, auth testa credenziali di default, brute fa brute force.
OID (Object Identifier)
Identificatore numerico gerarchico per oggetti SNMP. Formato: numeri separati da punti (es: 1.3.6.1.2.1.1.1.0 = sysDescr). Leggi un OID per ottenere quella specifica informazione dal dispositivo.
Penetration Testing
Attività professionale autorizzata di test sicurezza che simula un attacco reale per trovare vulnerabilità. Distinto dall'"hacking" non autorizzato per il consenso esplicito del proprietario del sistema.
PIN (SIP)
Codice numerico (tipicamente 4-6 cifre) che autentica un account SIP. I PIN deboli (1234, 0000) permettono a chiunque di registrare quell'estensione e effettuare chiamate.
Pipeline
In exa-dune: sequenza automatizzata di fasi (scan → analisi → attacchi → post-exploit → report). In Unix: concatenazione di comandi con | dove l'output di uno diventa input del successivo.
Port Scanning
Processo di sondare un dispositivo su tutte le porte per determinare quali sono aperte e quali servizi le usano. Punto di partenza di ogni penetration test.
Post-Exploitation
Fase dopo aver ottenuto accesso: raccolta di informazioni, mantenimento dell'accesso, escalation privilegi, pivot verso altri sistemi. Dimostra l'impatto reale della vulnerabilità.
RCE (Remote Code Execution)
Vulnerabilità critica che permette di eseguire codice arbitrario su un sistema remoto senza autenticazione. CVSS spesso 9.0-10.0. La vulnerabilità più pericolosa esistente.
SecLists
Raccolta open source di wordlist per sicurezza: password, username, percorsi web, payload. Su Kali: /usr/share/seclists/. exa-dune le usa se disponibili, altrimenti usa le wordlist integrate.
SIP (Session Initiation Protocol)
Protocollo standard RFC 3261 per comunicazioni multimediali (VoIP). Gestisce la segnalazione: chi chiama chi, negoziazione codec, avvio/fine chiamata. Porta default: 5060/UDP.
Stealth Scan
Scansione "furtiva" che riduce il numero di pacchetti e la velocità per evitare il rilevamento da IDS/IPS e firewall. In exa-dune: attivato con --stealth, usa timing T2 e 1 thread.
TLS/SSL
Protocolli crittografici per comunicazioni sicure su Internet. SSL è obsoleto; TLS 1.2 e 1.3 sono attuali. TLS 1.0/1.1 sono deprecati e vulnerabili. HTTPS usa TLS su HTTP.
VoIP (Voice over IP)
Tecnologia che trasmette voce su reti IP invece delle tradizionali linee telefoniche. Usa protocolli come SIP per la segnalazione e RTP per il trasporto audio.
Wordlist
File di testo con una parola per riga, usato per attacchi dizionario. In exa-dune v2.4.0: 7 wordlist integrate (embedded) + 24 wordlist aggiuntive da SecLists per utenti, password, web, SNMP, LFI, DNS, credenziali IoT, RTSP, SIP.
Zone Transfer (AXFR)
Meccanismo DNS per replicare tutti i record di un dominio da un server a un altro. Se mal configurato, rivela l'intera struttura della rete interna a qualsiasi richiedente.
