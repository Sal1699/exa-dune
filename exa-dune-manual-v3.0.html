1. Introduzione e Concetti Base

1.1 Cos'è exa-dune?

exa-dune è un framework di penetration testing scritto in Bash (~11.700 righe), progettato per ambienti IoT e VoIP. Integra e coordina oltre 40 tool di sicurezza di Kali Linux attraverso un'interfaccia unificata, aggiunge 14 script ausiliari custom (Python, JavaScript, Shell), gestisce il ciclo completo di assessment e genera report PDF professionali in italiano con scoring CVSS 3.1.

Nota
Questo strumento è destinato esclusivamente a sistemi di tua proprietà o per i quali hai esplicita autorizzazione scritta. L'uso non autorizzato è illegale.

1.2 Cosa può fare exa-dune v3.0?

  • 51 subcomandi — dalla scansione alla post-exploitation
  • 14 script ausiliari — custom Python/JS/Shell per attacchi specializzati
  • Profili auto — voip-gateway, ip-camera, iot-adapter (lancia automaticamente i moduli giusti)
  • Propagazione credenziali — credenziali trovate vengono testate su tutti i servizi del target
  • Pipeline automatizzata — scan → attacco → post-exploit → PDF report con un solo comando
  • CVE database integrato — Hikvision, Dahua, Axis, Reolink, AudioCodes MP20x, Commend ET962
  • Analisi firmware — binwalk, credenziali hardcoded, binari SUID, certificati TLS
  • Storico SQLite — tutti gli assessment indicizzati e confrontabili nel tempo
  • Scheduling cron — assessment periodici automatici con diff automatico

4.2 Esempio con flag combinati

# Scan silenzioso con adaptive rate, timeout 30 min, output JSON
exa-dune scan 192.168.1.10 --stealth --adaptive --timeout 30 --json

# Dry-run per vedere cosa farebbe il modulo web
exa-dune web 192.168.1.10 --port 443 --scheme https --dry-run

4.3 Rate limiting adattivo (--adaptive)

Quando --adaptive è attivo, dopo ogni scan nmap exa-dune analizza il log cercando TCP reset e host irresponsivi:

  • Se >10% dei pacchetti riceve TCP reset → riduce NMAP_RATE_FAST del 20%
  • Se l'host risulta irresponsivo → passa automaticamente a -T2 (stealth)
  • Il rate minimo è 50 pps (fast) e 100 pps (full) per evitare blocchi completi

6. Ricognizione e Scansione

scan Port scan + fingerprinting + recon (nmap, arp-scan, searchsploit) 
exa-dune scan <TARGET> [--stealth] [--adaptive] [--udp-top N] [--no-full] [--vuln] [--timeout N]

Esegue in sequenza: ARP scan, nmap fast (top 1000 + script), nmap full (-p-), nmap UDP top N, searchsploit da XML nmap. Al termine mostra una tabella di attacchi consigliati basata sulle porte trovate.

FlagDefaultDescrizione
--stealthoffnmap -T2, rate ridotto
--adaptiveoffRiduce rate automaticamente
--udp-top N20Numero porte UDP da scansionare
--no-fulloffSalta nmap -p- (più veloce)
--vulnoffEsegui nmap --script vuln + searchsploit per banner

exa-dune network-sweep <CIDR> [--iface eth0] [--auto] [--timeout N]

Individua tutti gli host attivi, ne identifica il tipo (VoIP, camera, IoT, router) basandosi sulle porte aperte, e produce una tabella con il comando suggerito per ciascun host. Con --auto lancia automaticamente il profilo appropriato.

7. VoIP e Multimedia

sip SIP/VoIP enumeration (svmap, svwar, svcrack, nmap SIP NSE) 
exa-dune sip <TARGET> [--port 5060] [--range 100-500] [--transport udp|tcp]
  • svmap: fingerprint del server SIP (vendor, versione)
  • svwar: enumerazione extension nel range specificato
  • svcrack: brute force password per le extension trovate
  • Probe SIP manuale: REGISTER, OPTIONS, INVITE, INFO, SUBSCRIBE
  • Cattura digest challenge per crack offline
  • Toll fraud check: tenta chiamata senza autenticazione
  • sip_enum.py: user enumeration avanzata con timing analysis (se installato)
rtsp RTSP stream audit + Digest Auth brute force 
exa-dune rtsp <TARGET> [--port 554] [--wordlist FILE]
  • Enumera path RTSP (wordlist + path vendor-specific embedded)
  • Digest Auth brute force via Python (MD5 socket raw)
  • Snapshot capture con ffmpeg (proof-of-concept)
  • rtsp_fingerprint.py: identifica vendor da RTSP OPTIONS/DESCRIBE (17 vendor)
onvif ONVIF/gSOAP IP camera audit (WS-Security brute force) 
exa-dune onvif <TARGET> [--port 80] [--users FILE] [--pass FILE]
  • Probe anonimo: GetDeviceInformation, GetCapabilities, GetProfiles
  • Brute force WS-Security UsernameToken con digest SHA1
  • onvif_brute.py: testa prima 14 coppie default hardcoded, poi wordlist
voip-enum VoIP enumeration avanzata NUOVO 
exa-dune voip-enum <TARGET> [--port 5060] [--range 100-500] [--flood-test]
  • SIP OPTIONS flood: 100 richieste in 10s, verifica rate limiting (429)
  • User enumeration: distingue 401/403/404 per extension → identifica utenti validi
  • Toll fraud: SIP INVITE senza auth → [FOUND:CRITICAL] se accettato
  • Extension scan: svwar su range configurabile
  • TLS SIP: verifica porta 5061, analisi certificato
Attenzione
Il flood test può saturare dispositivi VoIP con hardware limitato. Usa --flood-test solo se necessario.

12. Script Ausiliari Python / JS / Shell

Installati in /usr/share/exa-dune/scripts/ tramite exa-dune install. Vengono richiamati automaticamente dai moduli principali quando disponibili, e possono essere usati in modo standalone.

12.1 Script Python

sip_enum.py — SIP User Enumeration

python3 sip_enum.py <TARGET> [--port 5060] [--range 100-500] [--method register|options] [--transport udp|tcp]

Enumera extension SIP via REGISTER/OPTIONS. Distingue 401 (utente valido), 403 (bloccato), 404 (non trovato). Include timing analysis: se la differenza di risposta tra utenti è >2 deviazioni standard segnala user enumeration via timing.

onvif_brute.py — ONVIF Default + Brute Force

python3 onvif_brute.py <TARGET> [--port 80] [--users FILE] [--passwords FILE]

Testa prima 14 coppie default hardcoded (admin:admin, admin:12345, Admin:admin...). Poi brute force con wordlist. Genera envelope SOAP WS-Security con UsernameToken + Nonce + SHA1 PasswordDigest. Se trova credenziali valide: esegue GetDeviceInformation e GetCapabilities.

modbus_rw.py — Modbus Read/Write PoC

python3 modbus_rw.py <TARGET> [--port 502] [--unit 1] [--action read|write|enum]

Implementa FC1-FC4 (read), FC5 (Write Single Coil), FC6 (Write Single Register), FC43 (Device Identification) via socket raw o pymodbus se installato. Il write scrive valore 0x0000 (safe per PoC). Segnala CRITICAL se il write riesce senza autenticazione.

coap_scan.py — CoAP Client Custom

python3 coap_scan.py <TARGET> [--port 5683] [--action discover|get|put|observe]

Implementa il protocollo CoAP manualmente (senza libcoap). Supporta: discovery via /.well-known/core, GET di risorse, PUT injection, OBSERVE per notifiche continue. Rileva risorse sensibili (/auth, /admin, /config).

rtsp_fingerprint.py — Vendor Fingerprint

python3 rtsp_fingerprint.py <TARGET> [--port 554]

Invia RTSP OPTIONS e DESCRIBE, analizza header Server: e Via:. Riconosce 17 vendor: Hikvision, Dahua, Axis, Reolink, AudioCodes, Commend, Bosch, Hanwha, Milestone, VxWorks e altri. Dopo l'identificazione testa i path RTSP specifici del vendor.

mqtt_audit.py — MQTT Deep Audit

python3 mqtt_audit.py <TARGET> [--port 1883] [--tls] [--user U] [--pass P]

Usa paho-mqtt. Testa accesso anonimo, subscribe su # e $SYS/#, analizza i messaggi ricevuti cercando pattern sensibili (password, token, key). Testa write su topic cmd/, set/, control/, actuator/.

upnp_exploit.py — UPnP IGD Exploit

python3 upnp_exploit.py <TARGET> [--action discover|addport|getip|enum]

Discovery SSDP, GetExternalIPAddress (rivela IP WAN), AddPortMapping senza autenticazione. Cleanup automatico con DeletePortMapping dopo il test.

cam_cve_batch.py — CVE Batch Scanner

python3 cam_cve_batch.py <IP_LIST_O_CIDR> [--vendor auto] [--output results.json]

Accetta CIDR o file con lista IP. Per ogni host: fingerprinta il vendor, testa tutti i CVE applicabili (Hikvision, Dahua, Axis, Reolink, AudioCodes, Commend). Output tabella + JSON opzionale.

http_iot_fuzz.py — HTTP Fuzzer Embedded

python3 http_iot_fuzz.py <TARGET> [--port 80] [--scheme http|https] [--mode paths|inject|bypass|all]

60+ CGI path comuni per device embedded. Injection via GET con payload |id, ;id, $(id). Auth bypass via header manipulation (X-Forwarded-For, X-Real-IP). Lista default creds IoT per HTTP Basic Auth.

sip_pcap_creds.py — SIP PCAP Credential Extractor

python3 sip_pcap_creds.py <PCAP_FILE> [--wordlist FILE] [--output creds.txt]

Usa scapy per leggere PCAP. Estrae header Authorization: Digest dai pacchetti SIP. Tenta crack offline del hash MD5 (username:realm:password format). Output in formato hashcat 11400 per crack avanzato.

12.2 Script JavaScript (Node.js)

websocket_audit.js — WebSocket Auditor

node websocket_audit.js <TARGET> [--port 8080] [--path /ws] [--mode probe|fuzz|auth]

Testa connessioni WebSocket su 14 path comuni (/ws, /live, /stream, /api/ws...). Modalità probe: cattura messaggi iniziali. Modalità auth: verifica se i dati vengono inviati senza token. Modalità fuzz: invia JSON malformati. Supporta ws:// e wss://.

xss_generator.js — XSS Payload Generator

node xss_generator.js [--target URL] [--param field] [--mode test|steal|exec]

Genera payload XSS in 5 encoding diversi (HTML entities, URL, unicode, hex, base64). Con --target testa direttamente la reflection via HTTP. Include test specifico per CVE-2019-9955 (AudioCodes XSS su /Forms/login).

12.3 Script Shell

voip_capture.sh — VoIP Traffic Capture

voip_capture.sh [--iface eth0] [--target 192.168.1.0/24] [--duration 60] [--output DIR]

Cattura traffico SIP (5060/5061 TCP+UDP) e RTP (10000-20000 UDP) con tcpdump. Post-capture: analisi con tshark/sngrep, estrazione stream RTP se disponibile rtpdump, estrazione automatica credenziali SIP via sip_pcap_creds.py.

net_recon.sh — Quick Network Recon

net_recon.sh <CIDR> [--output DIR]

arp-scan per host discovery, nmap ping sweep parallelo, nmap top-50 ports per ogni host vivo, fingerprinting automatico tipo device. Output compatibile con exa-dune network-sweep.

13. Pipeline Automatizzata

exa-dune pipeline <TARGET> [--profile PROFILO] [--from-scan DIR] [--dry-run]

La pipeline esegue automaticamente tutte le fasi:

FASE 1

Scansione

nmap fast + full + UDP. Produce nmap_fast.nmap con porte aperte.

FASE 2

Auto-suggest

Analizza porte aperte e mappa a moduli exa-dune con priorità. Mostra tabella attacchi consigliati.

FASE 3

Attacchi in ordine di priorità

Esegue i moduli in sequenza: prima i più critici (telnet, SIP, SMB), poi gli altri.

FASE 4

Propagazione credenziali

Tutte le credenziali trovate vengono testate cross-service automaticamente.

FASE 5

Report HTML + PDF

Genera report professionale con executive summary, CVSS 3.1, findings table, credenziali trovate. PDF salvato su Desktop.

# Pipeline completa con profilo voip-gateway
exa-dune pipeline 192.168.1.10 --profile voip-gateway

# Riusa scan esistente (salta fase 1)
exa-dune pipeline 192.168.1.10 --from-scan /root/pentest/exa-dune-192-168-1-10_20260322/

# Dry-run per vedere cosa farebbe
exa-dune pipeline 192.168.1.10 --dry-run

14. Orchestrazione: resume, schedule, history, update

resume Riprende assessment interrotto NUOVO 
exa-dune resume <DIR_SESSIONE>

Legge session.json nella directory, mostra i moduli già completati e riprende dall'ultimo punto. Ogni modulo completato aggiorna automaticamente session.json.

# Assessment interrotto dopo scan e sip
exa-dune resume /root/pentest/exa-dune-192-168-1-10_20260322/
# Output: "Moduli completati: scan, sip (2/8). Riprendo da: snmp..."
schedule Scan periodici via cron NUOVO 
exa-dune schedule --add --target <IP> --cron "EXPR" --profile <PROFILO>
exa-dune schedule --list
exa-dune schedule --remove <ID>

Aggiunge cron job che esegue automaticamente la pipeline sul target. Dopo ogni run schedulata, esegue exa-dune diff contro la run precedente e salva il report differenziale.

# Assessment settimanale ogni lunedì alle 2:00
exa-dune schedule --add --target 192.168.1.10 --cron "0 2 * * 1" --profile voip-gateway

# Lista job attivi
exa-dune schedule --list
history Storico assessment SQLite NUOVO 
exa-dune history [--target IP] [--last N] [--findings ID]

Tutti gli assessment vengono salvati automaticamente in /root/pentest/exa-dune.db (SQLite). Tabella assessments: target, modulo, timestamp, directory, conteggi finding. Tabella findings: ogni singolo finding con severità e messaggio.

# Ultimi 10 assessment
exa-dune history --last 10

# Tutti gli assessment sul target
exa-dune history --target 192.168.1.10

# Finding di un assessment specifico
exa-dune history --findings 42
update Aggiornamento automatico NUOVO 
exa-dune update

Trova il repo git locale dello script, esegue git pull origin main, copia l'aggiornamento in /usr/local/bin/exa-dune, aggiorna anche gli script ausiliari. Se il repo non è trovato, scarica direttamente da GitHub raw.

15. Report CVSS 3.1 — Formato Italiano

exa-dune report <DIR_SESSIONE> [--pdf] [--open]

15.1 Struttura del report

  1. Copertina — Target, data, classificazione CONFIDENZIALE
  2. Sommario Esecutivo — Testo narrativo auto-generato: "Il target X espone Y vulnerabilità critiche..."
  3. Conteggio per severità — CRITICAL / HIGH / MEDIUM / LOW / INFO con contatori
  4. Tabella Finding — ID, Severità, CVSS Score, Titolo, Descrizione
  5. Dettaglio CRITICAL/HIGH — Descrizione estesa + raccomandazione per ogni finding grave
  6. Credenziali Trovate — Tabella con service, host, porta, user, password, source
  7. Appendice — Output raw dei tool (collassabile)
